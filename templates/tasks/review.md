# 任务复盘

本文件按条目追加复盘。每条复盘必须有稳定的 `Review ID`，方便后续 AI 或人工按 ID、标签、影响范围快速检索。

## 索引

| Review ID | 日期 | 类型 | 范围 | 状态 | 标签 |
| --- | --- | --- | --- | --- | --- |

## 模板

```md
## REV-YYYYMMDD-001 标题

Meta:
- Date: YYYY-MM-DD
- Type: feature / bugfix / refactor / docs / chore
- Scope: [changed area]
- Status: done / partial / blocked
- Tags: tag1, tag2
- Related files:
  - `path/to/file`

### Summary

[一句话说明这次做了什么]

### Root Cause

[为什么需要做]

### Fix

[如何处理]

### Verification

[跑了什么检查，结果如何]

### Risk

[剩余风险]

### Follow-up

[后续观察或待办]
```
