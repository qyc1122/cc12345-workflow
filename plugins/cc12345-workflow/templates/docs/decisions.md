# 架构决策记录

此文件用于记录重要但不紧急、会长期影响项目结构或协作边界的架构决策。

## 记录方式

每条 ADR 追加到本文末尾。新增前先检查已有最大编号，使用下一个四位递增编号（如 `0001`、`0002`）；已使用编号不得复用。

```md
## 0001. 决策标题

### 状态

Proposed / Accepted / Superseded

### 背景

为什么需要做这个决策。

### 决策

最终选择的方案。

### 备选方案

考虑过但没有采用的方案，以及原因。

### 影响

正向影响、风险、迁移成本、验证方式。
```

只记录已经形成判断的架构决策，不把临时任务进展或未经确认的项目事实写成架构决策。

## 不适合记录为决策的内容

- 单次 bug 修复、临时排查结论或任务进展。
- 尚未验证的猜测、个人偏好或一次性实现取舍。
- 只影响当前任务的待办、风险和复盘结论；这些应写入 `tasks/todo.md` 或 `tasks/review.md`。
- 用户纠正 AI 后形成的通用工作方式；这些应写入 `tasks/lessons.md`。
